# GEMINI.md — AI 開発エージェント行動原則

このファイルは Gemini CLI / Codex など、AI 開発エージェントが本リポジトリで作業する際の行動規範を定義する。

## 基本方針

- 最優先ゴール：短期キャッシュ回収
- 正確性・拡張性・美しさは副次
- 要件定義書・`REQUIREMENTS.md` に絶対従う

## 3つのモード

### 1. 設計モード

- 既存要件の整理のみ行う
- 新機能提案は禁止
- 不明点は `{{TODO: 未確定}}` として残す

### 2. 実装モード

- 5画面構成を絶対に壊さない
- 1変更 = 1意図
- 不要な抽象化・共通化は禁止

### 3. レビューモード

- 「回収効率を下げていないか」を最優先で確認
- UX 向上という名の親切化は NG

## 禁止事項（厳守）

- コマンドの自動実行
- 破壊的操作（DB初期化、`rm -rf` 等）
- 要件にない機能追加
- 説明文の増量
- 画面・導線の追加

## 実行コマンドについて

- すべて例示のみ
- 実行前に必ずユーザー確認を挟む
- 環境変数や秘密情報は生成・コミットしない

## 判断に迷ったら

次の問いで判断する：

> それは「¥980を考えずに払わせる」助けになるか？

NO なら実装しない。
